## 1. 背景与目标
- 对应议题：WS-30《[前端] OctoLink 控制台多语言 (i18n) 支持:默认中文,可选英文》。
- 需求来源:老板(王祺)直接指示,控制台前端需要多语言能力,**默认中文 (zh-CN)**,**可选英文 (en)**,首次/无偏好时落到中文。
- 目标:为 `frontend/`(Next.js 15 Pages Router / React 18 / MUI 5)引入 i18n 基础设施,提供顶部语言切换器(持久化),并把核心交互路径的可见文案外置到 zh / en 两份 message catalog。
- 范围(MVP):基础设施 + 语言切换 + 核心路径(登录、侧边导航、顶栏、账户菜单、设备/凭据/通知/账户/设置页头)。长尾文案标注 TODO 增量补齐(见 §6)。

## 2. 设计思路与方案选型
### 2.1 框架选型:react-i18next(客户端,无 URL 前缀)
当前前端无任何 i18n 依赖,且 `_app.js` 在 `useEffect` 中 `setTheme` 后才渲染应用(`theme && (...)`),即整站为**纯客户端渲染外壳**。因此选择:

- **react-i18next + i18next + i18next-browser-languagedetector**,纯客户端初始化。
- **不使用 `next-i18next` 的 locale 路由(`/en` 前缀)**:避免改动现有路由结构与鉴权 guard(`src/guards`),按议题建议走"无前缀 + cookie/localStorage 持久化"方案。
- catalog 以 JS `import` 直接打包(无 HTTP backend),SSR/构建期安全。
- 因整站客户端渲染,服务端静态产物为统一外壳,**不存在 hydration 文案不一致**问题。

### 2.2 语言与回退
- `supportedLngs: ['zh', 'en']`,`fallbackLng: 'zh'`,`load: 'languageOnly'` + `nonExplicitSupportedLngs`,使 `zh-CN`/`zh-*` 归一到 `zh` catalog。
- 缺失 key 兜底:`fallbackLng` 回退默认语言;`react-i18next` 默认在 key 缺失时返回 key 字符串,**不出现空白/报错**。
- 默认中文:无任何持久化偏好时 detector 命中 navigator,再回退 `zh`;`_document.js` 的 `<Html lang="zh">` 与 `<html lang>` 运行时同步。

### 2.3 持久化
- `i18next-browser-languagedetector`:`order: ['localStorage', 'cookie', 'navigator']`,`caches: ['localStorage', 'cookie']`,key = `octolink.lang`。
- 切换语言即写入 localStorage + cookie,刷新/重进不丢失。

### 2.4 换标一致性
- 品牌名 **OctoLink** 不进可翻译文案,保持原样硬编码(catalog 中仅作为固定串拼接在标题里,不被错翻),与换标工作 (WS-27) 解耦。

## 3. 接口 / 文件结构
```
frontend/src/i18n/
  index.js                 # i18next 初始化(detector + react binding + 配置)
  locales/zh.json          # 中文 catalog(默认)
  locales/en.json          # 英文 catalog
frontend/src/components/
  language-switcher.js      # 顶栏语言切换控件(IconButton + Menu)
```

### 3.1 对外 API
- `import i18n from 'src/i18n'`:已初始化的 i18next 实例。
- `import { SUPPORTED_LANGUAGES, DEFAULT_LANGUAGE, LANGUAGE_STORAGE_KEY } from 'src/i18n'`。
- 组件内使用 `const { t } = useTranslation()`,调用 `t('nav.devices')`。
- 富文本(含链接/加粗)使用 `<Trans i18nKey="login.openSource" components={{ gh: <Link .../> }} />`。

### 3.2 catalog 命名空间(扁平分组)
`language` / `nav` / `topnav` / `account` / `login` / `notifications` / `settings` / `credentials` / `devices` / `overview`。

## 4. 部署 / 使用
- 安装依赖(已写入 `package.json`):`i18next@^23`、`react-i18next@^14`、`i18next-browser-languagedetector@^8`。
  ```bash
  cd frontend && npm install
  ```
- 本地开发:`npm run dev`;生产构建:`npm run build && npm run start`。
- 使用方式:登录后顶栏("地球"图标)下拉选择 中文 / English,选择即持久化。
- 新增文案:在 `locales/zh.json` 与 `locales/en.json` 同步增加同名 key,组件内用 `t('group.key')` 引用。
- ⚠️ 上线节奏:本任务仅落地代码库基础设施,**严禁未经老板审批直接上线部署**(并入换标/MVP 收尾统一处理)。

## 5. 测试 / 验收
- ✅ `npm run build` 通过,19 个页面静态产物全部生成,无类型/编译错误。
- ✅ 默认渲染中文;顶栏切换器可切到英文并持久化(localStorage `octolink.lang` + cookie)。
- ✅ 缺失 key 回退到默认语言/key,无空白报错。
- ✅ 未改动路由/鉴权 guard,现有功能与构建不受影响。
- 已覆盖核心交互路径:登录页、侧边导航(全部菜单)、顶栏(通知/源码/语言 tooltip)、账户菜单(账户/退出登录)、通知中心、设备页头、凭据页(标题/提示/用户名密码标签)、账户页、设置页、总览页标题。

### 验收手测建议
1. 清空浏览器存储后首次访问 → 控制台中文渲染。
2. 顶栏切换 English → 导航/登录等核心文案变英文;刷新后保持英文。
3. 再切回中文 → 立即生效并持久化。
